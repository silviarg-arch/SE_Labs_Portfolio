\# Lab 2 – UML Use Case Diagrams



\## Description

In this lab, we created use case diagrams to represent system functionalities and user interactions.



\## ATM System

The ATM system includes a Customer actor interacting with functions such as withdrawing money, checking balance, and validating the PIN. The diagram highlights essential actions and system responses.



\## Medical Clinic System

This system involves Patient and Doctor actors. Patients can book appointments and view records, while doctors conduct consultations and update medical data. Relationships such as <<include>> and <<extend>> were used to show required and optional actions.



\## Car Insurance System

The system models interactions between the Customer and the insurance service, including creating policies, submitting claims, and processing payments. The diagram focuses on service-related functionalities and user interactions.
