\# Lab 3 – UML Class Diagrams



\## Description

In this lab, we modeled the internal structure of systems using class diagrams.



\## ATM System

The system includes classes such as Customer, ATM, BankAccount, and Transaction. Key relationships include associations between customer and account, and dependencies between ATM and bank services.



\## Medical Clinic System

The main classes are Patient, Doctor, Appointment, MedicalRecord, and Treatment. Important relationships include composition between Patient and MedicalRecord, and aggregation between MedicalRecord and Treatment.



\## Car Insurance System

The system includes classes such as Customer, Policy, Claim, and Payment. Associations represent ownership of policies, while dependencies show how claims and payments interact with the system.
